# UC-004: TUI Application

## Characteristic Information
**ID:** UC-004  
**Name:** TUI Application  
**Priority:** High  
**Actor:** User

## Description
Build the complete Textual TUI application with layout, metrics display, progress bars, colors, keyboard controls, and auto-refresh functionality.

## Main Success Scenario
1. Initialize Textual application with `MonitorApp` class
2. Create layout with sections:
   - GPU metrics (VRAM progress bar, temperature, fan speed)
   - RAM metrics (progress bar)
   - Git metrics section (uncommitted lines, commit status, time since last commit)
   - Recent commits scrollable list (last 5)
3. Render with colors and spacing
4. Handle keyboard input:
   - `q`: Quit application
   - `r`: Refresh data immediately
   - `h`: Show help overlay
   - `space`: Pause/resume auto-refresh
5. Auto-refresh data every 5 seconds
6. Adapt layout to terminal resize

## Extensions
- 1a. Terminal too small → show truncated or scrollable content
- 1b. GPU not available → hide GPU section or show placeholder
- 1c. Not in Git repo → hide Git section
- 1d. Data collection fails → keep previous data, show error indicator

## Sub-Variations
- Large terminal (full display)
- Small terminal (truncated display)
- Dark/light terminal themes
- Pause/resume state
- Refresh in progress

## Related Information
- **Textual widgets:**
  - `Static` for text labels and metrics
  - `Progress` for VRAM/RAM usage
  - `ListView` or `RichLog` for recent commits
  - `Overlay` for help dialog
  - `Column` and `Horizontal` for layouts
- **Color palette:** green=good (<70%), yellow=warning (70-85%), red=critical (>85%)
- **Timer:** `set_interval(5, refresh_data)`
- **Key handling:** Override `on_key` or handle `Key` messages
- **Resize handling:** Override `on_resize` or use responsive layouts
